# Keycloak 사용자 조회 스킬

이 스킬은 스타-클라우드의 Keycloak 인증 서버에 등록된 사용자 목록을 조회합니다. PostgreSQL 데이터베이스에 직접 접속하여 Keycloak의 `user_entity` 테이블에서 사용자 정보를 가져옵니다.

## 사용 시나리오

이 스킬은 다음 상황에서 사용하세요:
- 스타-클라우드에 등록된 사용자 목록을 확인하고 싶을 때
- Keycloak에 어떤 사용자 계정이 있는지 확인하고 싶을 때
- 사용자 계정 상태(활성화/비활성화)를 확인하고 싶을 때
- 새로운 사용자 등록 후 확인하고 싶을 때

## 데이터베이스 연결 정보

스타-클라우드 환경에서는 다음과 같은 데이터베이스 정보를 사용합니다:
- **컨테이너 이름**: `startcloud-postgres`
- **데이터베이스**: `keycloak`
- **사용자명**: `startcloud`
- **비밀번호**: `startcloud_secret`

## 쿼리 실행 방법

사용자 정보를 조회하려면 다음 SQL 쿼리를 실행하세요:

```sql
SELECT 
    username, 
    email, 
    first_name, 
    last_name, 
    enabled, 
    to_char(to_timestamp(created_timestamp/1000), 'YYYY-MM-DD HH24:MI:SS') as created_date 
FROM user_entity 
ORDER BY created_timestamp;
```

## 실행 단계

1. **PostgreSQL 컨테이너에 접속**: `startcloud-postgres` 컨테이너에 접속합니다.
2. **SQL 쿼리 실행**: 위의 SQL 쿼리를 실행하여 사용자 정보를 조회합니다.
3. **결과 포맷팅**: 결과를 읽기 쉬운 표 형식으로 포맷팅합니다.
4. **요약 정보 제공**: 총 사용자 수, 활성 사용자 수, 최근 생성된 사용자 등 요약 정보를 제공합니다.

## 출력 형식

결과는 다음과 같은 형식으로 제공하세요:

### 🔍 Keycloak에 등록된 사용자 목록

현재 Keycloak에 총 **X명의 사용자**가 등록되어 있습니다:

#### 1. 사용자 계정 1
- **사용자명**: [username]
- **이메일**: [email]
- **이름**: [first_name]
- **성**: [last_name]
- **상태**: 활성화됨/비활성화됨 (enabled: t/f)
- **생성일**: [created_date]
- **역할**: [역할 설명]

#### 2. 사용자 계정 2
...

### 📊 요약
- **총 사용자 수**: X명
- **활성 사용자**: Y명
- **일반 사용자**: Z명
- **시스템 계정**: W명
- **가장 최근 생성**: [username] ([created_date])

## 주의사항

1. **데이터베이스 접근**: 이 스킬은 PostgreSQL 데이터베이스에 직접 접근하므로 데이터베이스 컨테이너가 실행 중이어야 합니다.
2. **권한**: 적절한 데이터베이스 사용자 권한이 필요합니다.
3. **민감한 정보**: 사용자 정보는 민감할 수 있으므로 결과를 안전하게 처리하세요.

## 예시 응답

사용자가 "Keycloak에 스타-클라우드에 등록된 사용자가 누가 있어?"라고 질문하면:

1. PostgreSQL 컨테이너에 접속하여 쿼리를 실행합니다.
2. 결과를 표 형식으로 포맷팅합니다.
3. 사용자 목록과 요약 정보를 제공합니다.

## 문제 해결

- **컨테이너 접속 실패**: `startcloud-postgres` 컨테이너가 실행 중인지 확인하세요.
- **쿼리 실패**: 데이터베이스 연결 정보가 올바른지 확인하세요.
- **결과 없음**: `user_entity` 테이블에 데이터가 없는 경우입니다.
